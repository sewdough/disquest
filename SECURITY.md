# Security Policy

## Scope
This project is a local, client-side utility.
It does not operate any servers, collect user data, or handle credentials.

Security reports should be limited to:
- Arbitrary code execution
- Dependency vulnerabilities
- Unexpected file system access
- Process or permission escalation
- Crashes or behavior that could cause data loss

Reports related to third-party platform terms of service, usage policies,
or account enforcement are **out of scope**.

## Supported Versions
This project is provided as-is.
Only the latest commit on the default branch is considered current.

There is no guarantee of long-term maintenance.

## Reporting a Vulnerability
If you believe you have discovered a legitimate security issue,
please report it **privately**.

Do not open a public issue.

You may report via:
- GitHub Security Advisories (preferred, if available)
- Direct contact through the repository ownerâ€™s GitHub profile

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Relevant logs or error output
- Your environment (OS, Node.js version, etc.)

## Disclosure
This project follows responsible disclosure on a best-effort basis.
If a report is deemed valid:
- The issue may be fixed or mitigated
- A note may be added to the repository or release history

There is no bug bounty program.
